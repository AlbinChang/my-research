# docs/index.html 自动发现文档机制说明

> 版本：2026-05-30 · 仓库：`AlbinChang/my-research` · 分支：`master`

---

## 1. 背景与动机

`docs/` 目录是 GitHub Pages 的发布根目录，`docs/index.html` 是站点入口。随着文档持续增加（调研报告、PPT 演示、技术手册等），手动维护 `index.html` 中的文件列表变得不可持续——每新增一个 `.md` 或 `.html` 文件都需要修改 HTML 源码。

**核心目标**：`index.html` 必须能够**自动发现** `docs/` 目录下所有 `.md` 和 `.html` 文件，无需任何人工介入。

---

## 2. 架构概览

自动发现引擎采用**三级递进策略**，按优先级从高到低依次尝试：

```
┌──────────────────────────────────────────────┐
│              浏览器加载 index.html              │
└──────────────────┬───────────────────────────┘
                   │
                   ▼
         ┌─────────────────┐
         │  Tier 1: 本地清单  │  ← fetch('files.json')
         │  docs/files.json  │    最高优先级，预生成 JSON
         └────────┬──────────┘
                  │ 失败 / 404
                  ▼
         ┌─────────────────┐
         │  Tier 2: 浏览器缓存 │  ← localStorage
         │  localStorage     │    30 分钟 TTL
         └────────┬──────────┘
                  │ 过期 / 不存在
                  ▼
         ┌─────────────────┐
         │  Tier 3: 在线兜底  │  ← GitHub Git Trees API
         │  GitHub API       │    recursive=1 递归获取
         └─────────────────┘
```

| 层级 | 数据源 | 延迟 | 适用场景 |
|------|--------|------|----------|
| Tier 1 | `docs/files.json` | 极低（本地文件） | 正常运行，最推荐 |
| Tier 2 | `localStorage` 缓存 | 极低（内存） | files.json 丢失时的快速回退 |
| Tier 3 | GitHub API | 中等（网络） | 本地无清单且无缓存时的最终兜底 |

## 3. files.json — 本地预生成清单

### 3.1 文件格式

`docs/files.json` 是一个 UTF-8 编码的 JSON 文件，由 `refresh-docs.ps1` 生成。结构如下：

```json
{
  "generated": "2026-05-30T08:40:02+08:00",
  "generator": "refresh-docs.ps1",
  "basePath": "docs",
  "files": [
    {
      "name": "Agent_Skills全面深度调研报告.md",
      "path": "agent/Agent_Skills全面深度调研报告.md",
      "type": "md",
      "dir": "agent",
      "title": ""
    },
    ...
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `generated` | ISO 8601 时间戳 | 清单生成时间，用于与 localStorage 缓存做新旧比较 |
| `generator` | 字符串 | 生成工具标识（固定为 `refresh-docs.ps1`） |
| `basePath` | 字符串 | 基础路径（固定为 `docs`） |
| `files` | 数组 | 文件条目列表，按目录名和文件名排序 |
| `files[].name` | 字符串 | 文件名（含扩展名） |
| `files[].path` | 字符串 | 相对于 `docs/` 的路径 |
| `files[].type` | `"md"` 或 `"html"` | 文件类型，由扩展名自动判定 |
| `files[].dir` | 字符串 | 所属一级子目录名（根目录下文件为空字符串） |
| `files[].title` | 字符串 | 预留的自定义标题字段，当前为空 |

### 3.2 兼容性

`index.html` 同时兼容旧格式（纯数组），通过 `normalizeManifest()` 函数统一标准化：

```javascript
function normalizeManifest(manifest) {
    if (Array.isArray(manifest)) {
        return manifest.map(normalizeFile);        // 旧格式：纯数组
    }
    if (manifest && Array.isArray(manifest.files)) {
        return manifest.files.map(normalizeFile);  // 新格式：包含元数据
    }
    throw new Error('files.json 格式不正确');
}
```

## 4. refresh-docs.ps1 — 清单生成脚本

### 4.1 位置与用途

脚本位于仓库根目录 `refresh-docs.ps1`，作用是在**任何新增或删除文档后**，重新扫描 `docs/` 并生成最新的 `files.json`。

### 4.2 工作流程

```
refresh-docs.ps1
    │
    ├─ 定位 docs/ 目录
    │
    ├─ 递归扫描所有 .md 和 .html 文件
    │    自动排除：index.html、files.json
    │
    ├─ 逐文件提取元数据
    │    ├─ 相对路径（将 \ 统一转为 /）
    │    ├─ 扩展名 → type（md / html）
    │    └─ 一级子目录 → dir
    │
    ├─ 组装 JSON 清单（含 generated 时间戳）
    │
    └─ 输出到 docs/files.json（UTF-8 编码）
```

### 4.3 运行方式

在仓库根目录执行：

```powershell
.\refresh-docs.ps1
```

输出示例：

```
扫描 docs/ 目录…

=== 清单已生成 ===
输出文件 : D:\调研探索\docs\files.json
专题分类 : 4
文档总数 : 21
Markdown : 17
HTML     : 4

提示：index.html 已配置为自动读取此 files.json。
      将本项目推送到 GitHub Pages 后，即使没有 files.json，
      index.html 也会通过 GitHub API 自动发现所有文件。
```

> **注意**：脚本会自动排除 `index.html` 和 `files.json` 自身，避免索引页和清单文件出现在文档列表中。

## 5. index.html 自动发现引擎详解

### 5.1 入口函数 `discoverFiles()`

```javascript
function discoverFiles() {
    // 1) 尝试 files.json（本地预生成清单），优先级最高
    return fetch('files.json', { cache: 'no-cache' })
        .then(function(resp) {
            if (!resp.ok) throw new Error('files.json not available');
            return resp.json();
        })
        .then(function(manifest) {
            var files = normalizeManifest(manifest);
            var manifestTs = manifest.generated ? new Date(manifest.generated).getTime() : 0;
            var cached = tryLoadCache(manifestTs);
            if (cached) {
                return cached;  // 缓存不早于 files.json，直接使用
            }
            saveCache(files);  // files.json 更新，刷新缓存
            return files;
        })
        .catch(function() {
            // 2) files.json 不可用，尝试 localStorage 缓存
            var cached = tryLoadCache(0);
            if (cached) return Promise.resolve(cached);
            // 3) GitHub API 兜底
            return fetchFromGitHub();
        });
}
```

### 5.2 Tier 1: files.json 本地清单（最高优先级）

- 通过 `fetch('files.json', { cache: 'no-cache' })` 加载
- 缓存策略设为 `no-cache`，确保始终获取最新版本（但仍可利用 304 协商缓存）
- 成功后提取 `generated` 时间戳，与 localStorage 比较：
  - 若 localStorage 缓存时间 **不早于** files.json 生成时间 → 直接使用缓存（秒开）
  - 否则 → 使用 files.json 数据并刷新缓存

### 5.3 Tier 2: localStorage 浏览器缓存

当 `files.json` 不可用（404、网络错误等）时，回退到浏览器本地缓存。

**缓存结构**：

```json
{
  "ts": 1717051200000,      // 毫秒时间戳：缓存写入时刻
  "files": [ ... ]          // 文件列表（与 files.json 同构）
}
```

**缓存策略**（`tryLoadCache(manifestTs)`）：

| 场景 | 判断条件 | 结果 |
|------|----------|------|
| 有 files.json 时间戳 | `cache.ts >= manifestTs` | 缓存有效，直接返回 |
| 有 files.json 时间戳 | `cache.ts < manifestTs` | 缓存过期，使用 files.json |
| 无 files.json 时间戳 | `now - cache.ts < 30分钟` | 缓存有效，直接返回 |
| 无 files.json 时间戳 | `now - cache.ts >= 30分钟` | 缓存过期，进入 Tier 3 |

- **TTL**：30 分钟（`CACHE_TTL = 30 * 60 * 1000`）
- **存储键**：`docs_index_cache`
- **异常处理**：`localStorage` 配额溢出时静默失败，不影响主流程

### 5.4 Tier 3: GitHub API 在线兜底

当本地清单和缓存均不可用时，通过 GitHub Git Trees API 实时获取文件列表。

**API 端点**：

```
GET https://api.github.com/repos/{owner}/{repo}/git/trees/{branch}?recursive=1
```

**核心处理逻辑**（`fetchFromGitHub()`）：

```javascript
function fetchFromGitHub() {
    var apiUrl = 'https://api.github.com/repos/' + REPO_OWNER + '/' + REPO_NAME +
                 '/git/trees/' + REPO_BRANCH + '?recursive=1';

    return fetch(apiUrl, { cache: 'no-cache' })
        .then(function(resp) {
            if (resp.status === 403)
                throw new Error('GitHub API 速率限制，请稍后刷新');
            // ...
        })
        .then(function(data) {
            // 遍历 data.tree，过滤条件：
            // 1) 路径以 docs/ 开头
            // 2) 类型为 blob（文件，非 tree）
            // 3) 排除 index.html
            // 4) 扩展名为 .md 或 .html
            // ...
            saveCache(files);  // 成功后写入 localStorage
            return files;
        });
}
```

**API 特性**：

| 特性 | 说明 |
|------|------|
| 认证 | 无需 Token（公开仓库），但匿名访问有速率限制（60 次/小时/IP） |
| 递归 | `recursive=1` 一次请求获取全部目录树 |
| 403 处理 | 明确提示"速率限制"，建议稍后刷新 |
| 数据量 | 仓库 tree 中只过滤 `docs/` 下的 `.md`/`.html` blob |

## 6. 清单过期检测（后台异步）

页面渲染完成后，`checkFreshness()` 在后台异步调用 GitHub API，对比远程文件数量与当前渲染数量：

```
本地 files.json 文件数 < GitHub API 实际文件数
    │
    └─ 显示过期提示横幅（黄色）
       "检测到 GitHub 仓库中有 N 个文档，
        但当前清单仅包含 M 个（files.json 可能已过期）。"
       [从 GitHub API 刷新]  [关闭]
```

- 若用户点击"从 GitHub API 刷新"，直接调用 `fetchFromGitHub()` 覆盖当前数据
- 若 GitHub API 不可达（本地环境、网络限制），静默忽略，不打扰用户
- 刷新失败时显示错误提示横幅，自动回退到之前的数据

## 7. 渲染引擎

### 7.1 统计栏

页面顶部的统计栏实时展示：

| 指标 | DOM ID | 数据来源 |
|------|--------|----------|
| 专题分类数 | `statDirs` | `Object.keys(dirs).length` |
| 文档总数 | `statTotal` | `files.length` |
| Markdown 数 | `statMd` | `type === 'md'` 计数 |
| HTML 演示数 | `statHtml` | `type === 'html'` 计数 |

### 7.2 分区渲染

文件按一级子目录分组，每组独立渲染为一个 section：

```
┌──────────────────────────────────────┐
│ 🎬 电影                               │
│   2025 年美国高分电影推荐报告与深度分析   │
│  ┌──────────┐  ┌──────────┐          │
│  │ 🌐 幻灯片  │  │ 📝 文档   │          │
│  └──────────┘  └──────────┘          │
├──────────────────────────────────────┤
│ 🤖 Agent                              │
│   OpenAI Agent 技术调研、Skills 分析   │
│  ┌──────────┐  ┌──────────┐  ...     │
│  │ ...      │  │ ...      │          │
│  └──────────┘  └──────────┘          │
└──────────────────────────────────────┘
```

- **已知分类**（`SECTION_META`）：按预定义顺序排列，带自定义图标和描述
- **未知分类**：自动生成元数据（📁 图标 + 文件计数），追加到末尾

### 7.3 文件卡片

每个文件渲染为一张卡片，包含：

- **类型图标**：`🌐`（HTML 幻灯片）或 `📝`（Markdown 文档）
- **文件名**：优先使用 `title` 字段，否则去除扩展名并替换下划线
- **标签**：`幻灯片`（橙色）或 `文档`（绿色）
- **点击**：直接跳转到文件相对路径

## 8. 完整使用流程

### 场景 A：开发者新增文档后更新索引

```
1. 在 docs/ 下新增或删除 .md / .html 文件

2. 运行清单生成脚本：
   PS> .\refresh-docs.ps1

3. 提交并推送：
   PS> git add docs/files.json
   PS> git commit -m "更新文档清单"
   PS> git push
```

### 场景 B：GitHub Pages 访问者浏览文档

```
1. 访问 https://<user>.github.io/<repo>/
   （GitHub Pages 自动路由到 docs/index.html）

2. 浏览器加载 index.html

3. 自动发现引擎按优先级加载文件列表：
   ├─ 优先：docs/files.json（已预生成，秒开）
   ├─ 回退：localStorage 缓存（30 分钟有效）
   └─ 兜底：GitHub API（实时但可能受限速）

4. 渲染统计栏 + 按分类分区展示所有文档卡片

5. 后台异步检查 files.json 是否过期，过期则提示用户
```

### 场景 C：本地无 files.json 且无网络

```
优先级链逐级失败：
  files.json → 404
  localStorage → 无缓存（首次访问或已过期）
  GitHub API → 网络不可达

最终显示错误页面：
  "无法加载文档列表：...（本地无网络且未生成文件清单？
   请在仓库根目录运行 .\refresh-docs.ps1 生成 docs/files.json）"
```

## 9. 关键设计决策

### 9.1 为什么选 files.json 而非纯 GitHub API？

| 方案 | 优点 | 缺点 |
|------|------|------|
| files.json（当前） | 极低延迟、离线可用、无速率限制 | 需手动运行脚本更新 |
| 纯 GitHub API | 永远最新 | 网络延迟、匿名限速 60次/h、离线不可用 |
| 服务器端动态生成 | 零维护 | GitHub Pages 是纯静态托管，不支持服务端脚本 |

### 9.2 localStorage 缓存的设计目的

- **首次访问加速**：files.json 不可用时（如本地开发未预生成），缓存避免重复调 GitHub API
- **过期检测**：通过 `manifestTs` 比较实现精确的缓存失效，而非粗暴的时间 TTL
- **用户体验**：files.json 更新后首次访问秒开（Tier 1 命中，无需网络）

### 9.3 为什么排除 index.html？

`index.html` 是索引页自身，不应出现在文档列表中。同时 `files.json` 也被排除，避免清单文件自身被当作文档展示。

## 10. 故障排查

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 页面显示"正在发现文档…"后报错 | `files.json` 不存在且无缓存且 GitHub API 不可达 | 运行 `.\refresh-docs.ps1` 生成本地清单 |
| 新增文档未出现在列表中 | 未重新运行 `refresh-docs.ps1`，`files.json` 过期 | 运行 `.\refresh-docs.ps1` 并推送 |
| 显示"GitHub API 速率限制" | 匿名访问达到 60 次/小时上限 | 等待限制重置，或运行 `refresh-docs.ps1` 生成本地清单 |
| 黄色过期提示横幅 | `files.json` 落后于 GitHub 仓库实际文件数 | 点击"从 GitHub API 刷新"或重新运行脚本 |
| JSON 解析失败 | `files.json` 格式损坏或包含非法字符 | 重新运行 `.\refresh-docs.ps1` |

## 12. 扩展指南

### 12.1 新增自定义分类

编辑 `index.html` 中的 `SECTION_META` 对象（第 410-415 行）：

```javascript
var SECTION_META = {
    '电影': { icon: '🎬', title: '电影', subtitle: '...' },
    'agent': { icon: '🤖', title: 'Agent', subtitle: '...' },
    // 新增：
    'new-topic': { icon: '📊', title: '新专题', subtitle: '专题描述' }
};
```

只要 `dir` 值匹配 `docs/` 下的子目录名即可自动应用。

### 12.2 自定义文档标题

在 `files.json` 中修改对应文件的 `title` 字段，`index.html` 会优先显示 `title` 而非文件名。目前 `refresh-docs.ps1` 默认留空，可扩展脚本以支持从 Markdown 前置元数据或 HTML `<title>` 中提取。

### 12.3 支持更多文件类型

1. 在 `refresh-docs.ps1` 的 `Where-Object` 过滤条件中加入新扩展名
2. 在 `index.html` 的 `fetchFromGitHub()` 中同步修改正则 `\.(md|html)$/i`
3. 在渲染函数中为新型添加图标和标签样式

---

> **总结**：本自动发现引擎通过 `files.json` 预生成清单 + `localStorage` 浏览器缓存 + `GitHub API` 在线兜底的三级策略，实现了零人工维护的文档自动索引。在正常使用流程中，开发者只需在文档变更后执行一次 `.\refresh-docs.ps1`，其余完全自动化。
