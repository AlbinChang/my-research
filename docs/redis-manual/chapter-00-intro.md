# 序章：写在前面的话

## 0.1 为什么写这本书

我从 2016 年开始在生产环境中使用 Redis，经历了从 Redis 2.8 到 7.x 的整个演进过程。在这近 10 年的时间里，我亲眼见证了 Redis 从一个「缓存工具」蜕变为「高并发系统的核心基础设施」。

**几个让我印象深刻的真实场景：**

- **场景一**：某电商大促期间，核心商品详情页 QPS 从平时的 2 万飙升至 80 万，后端数据库直接被冲垮。引入 Redis 多级缓存后，数据库 QPS 从 80 万降到不足 1000，系统平稳度过大促。

- **场景二**：某社交应用的实时排行榜，每天数亿次分数更新，直接写 MySQL 导致写入延迟从 5ms 飙升到 2s。改用 Redis Sorted Set 后，写入延迟稳定在 1ms 以内。

- **场景三**：某金融系统的分布式锁由于未考虑 Redlock 的时钟漂移问题，导致同一笔订单被重复放行，造成数百万元资损。

这些场景不是个例。今天几乎所有的高并发系统都离不开 Redis，但「用起来简单，用对很难」。

市面上的 Redis 书籍和资料普遍存在两个极端：
1. **理论派**：大篇幅讲底层原理，到了实战环节就浅尝辄止
2. **碎片派**：博客式的零散知识点，缺乏系统性和深度

这本书试图走出第三条路——**以实战为主线，以问题为驱动，以案例为佐证**。每个知识点都回答三个问题：
- 它解决什么问题？
- 在生产中怎么用？
- 用不好会出什么事故？

## 0.2 目标读者

这本书的主要读者画像：

**👨‍💻 初级开发者（1-3年）**
- 会基本的 Redis 命令，但不知道什么时候该用什么数据结构
- 遇到过缓存穿透/击穿，但不知道如何系统性地解决
- 想学习高并发设计，但缺乏实战场景

**👩‍💻 高级开发者（3-8年）**
- 已经用 Redis 做过缓存、分布式锁等基础功能
- 遇到性能瓶颈或生产事故，想深入理解原理
- 需要设计高可用架构，但不确定选 Sentinel 还是 Cluster

**👨‍🔧 架构师 / 技术经理（8年+）**
- 需要设计高并发系统的整体缓存方案
- 关注 Redis 在微服务、容器化环境中的最佳实践
- 希望对团队进行 Redis 技术的系统性培训

## 0.3 前置知识要求

阅读本书你需要具备：

- **基础**：熟悉一种主流编程语言（本书示例以 Java 为主，但核心思想无关语言）
- **数据库**：了解关系型数据库的基本概念
- **网络**：了解 TCP/IP、HTTP 等基础网络协议
- **Redis**：如果你对 Redis 完全零基础，建议先花 30 分钟了解一下 `SET/GET` 等最基础的命令

不需要：
- ❌ 不需要深入理解 C 语言
- ❌ 不需要了解 epoll/select 的网络编程细节（虽然我们会讲原理，但不要求你写）
- ❌ 不需要系统学习过分布式系统理论

## 0.4 如何高效阅读

这本书的设计遵循「**渐进式深度**」原则：

```
第一层（1-2章）：基础实战 → 会用Redis处理高并发场景
第二层（3-5章）：原理与优化 → 理解背后机制，能排查问题
第三层（6-8章）：架构设计 → 能独立设计高并发缓存方案
第四层（9-11章）：运维与进阶 → 能承担Redis运维与架构演进
```

**推荐阅读策略：**

🔹 **方案A：系统学习（推荐）**
逐一阅读，每章读完后动手复现生产案例。预计耗时：3-5 天。

🔹 **方案B：问题驱动**
带着你现在遇到的真实问题来读：
- 慢查询问题 → 第五章
- 缓存一致性 → 第六章
- 分布式锁 → 第七章
- 限流需求 → 第八章

🔹 **方案C：团队培训**
将每章作为一个培训单元，每周一章，配合代码演练和案例讨论。

## 0.5 配套资源说明

本书所有代码示例均以以下环境验证：

```
Redis 版本: 6.2.14 / 7.2.4
Java 版本: OpenJDK 17
Redisson 版本: 3.27.0
Spring Boot 版本: 3.2.0
操作系统: CentOS 7.9 / macOS 14 / Windows 11
```

> **注意**：虽然示例基于特定版本，但核心原理和代码逻辑在所有 Redis 3.0+ 版本中均适用。
